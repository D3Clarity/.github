# Security Policy

**Do not report security vulnerabilities as public GitHub issues.**

If you discover a vulnerability in a D3Clarity repository:

1. **Preferred:** use **private vulnerability reporting** on the affected repository —
   the repo's **Security** tab → **Report a vulnerability** (GitHub private security advisories).
2. **Alternative:** email **info@d3clarity.com** with details and steps to reproduce.

Please include the affected repo, a description, reproduction steps, and impact.
We will acknowledge, triage privately, and coordinate a fix and disclosure.

Public issues are for non-sensitive work only. Security-related but non-sensitive
engineering work may be tracked as a normal issue with the `security` label.
